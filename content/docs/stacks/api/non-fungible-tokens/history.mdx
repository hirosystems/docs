<API>

<APIInfo method={"GET"} route={"/extended/v1/tokens/nft/history"}>

Retrieves all events relevant to a non-fungible token. This is useful when determining the ownership history of a particular asset.

More information on non-fungible tokens on the Stacks blockchain can be found [here](https://docs.stacks.co/write-smart-contracts/tokens#non-fungible-tokens-nfts).

### Query parameters

<Property required={true} deprecated={false} name={"asset_identifier"} type={"string"}>

Token asset class identifier

<span>Example:  `"SP2X0TZ59D5SZ8ACQ6YMCHHNR2ZN51Z32E2CJ173.the-explorer-guild::The-Explorer-Guild"`</span>

</Property>

<Property required={true} deprecated={false} name={"value"} type={"string"}>

Hex representation of the token's unique value

<span>Example:  `"0x0100000000000000000000000000000803"`</span>

</Property>

<Property required={false} deprecated={false} name={"limit"} type={"integer"}>

Max number of events to fetch

<span>Default:  `50`</span>

</Property>

<Property required={false} deprecated={false} name={"offset"} type={"integer"}>

Index of first event to fetch

<span>Example:  `42000`</span>

<span>Default:  `0`</span>

</Property>

<Property required={false} deprecated={false} name={"unanchored"} type={"boolean"}>

Whether or not to include events from unconfirmed transactions

<span>Example:  `true`</span>

<span>Default:  `false`</span>

</Property>

<Property required={false} deprecated={false} name={"tx_metadata"} type={"boolean"}>

Whether or not to include the complete transaction metadata instead of just `tx_id`. Enabling this option can affect performance and response times.

<span>Default:  `false`</span>

</Property>

| Status code | Description |
| :----------- | :----------- |
| `200` | Non-fungible token event history |

</APIInfo>

<APIExample>

```bash title="Terminal"
curl -X GET "https://api.mainnet.hiro.so/extended/v1/tokens/nft/history"
```
