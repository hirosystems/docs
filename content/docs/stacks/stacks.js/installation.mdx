Stacks.js is separated into many smaller packages, which can be installed individually and are published under the `@stacks` scope.

The most commonly used packages are:

<Accordions>
  <Accordion title="All available packages">
  [`@stacks/connect`](/stacks/stacks.js/packages/auth) - Stacks Connect, for building Stacks-ready web applications

  [`@stacks/auth`](/stacks/stacks.js/packages/auth) - Construct and decode authentication requests for Stacks apps.

  [`@stacks/transactions`](/stacks/stacks.js/packages/transactions) - Construct, decode transactions and work with Clarity smart contracts on the Stacks blockchain.

  [`@stacks/wallet-sdk`](/stacks/stacks.js/packages/wallet-sdk) - Library for building wallets, managing accounts, and handling keys for the Stacks blockchain.

  [`@stacks/storage`](/stacks/stacks.js/packages/storage) - Store and fetch files with Gaia, the decentralized storage system.

  [`@stacks/profile`](/stacks/stacks.js/packages/profile) - Functions for manipulating user profiles.

  [`@stacks/encryption`](/stacks/stacks.js/packages/encryption) - Encryption functions used by stacks.js packages.

  [`@stacks/network`](/stacks/stacks.js/packages/network) - Network and API library for working with Stacks blockchain nodes.

  [`@stacks/common`](/stacks/stacks.js/packages/common) - Common utilities used by stacks.js packages.

  [`@stacks/bns`](/stacks/stacks.js/packages/bns) - Library for interacting with the BNS contract.

  [`@stacks/stacking`](/stacks/stacks.js/packages/stacking) - Library for PoX stacking.

  [`@stacks/cli`](/stacks/stacks.js/packages/cli) - Command line interface to interact with auth, storage, and Stacks transactions.

  [`@stacks/blockchain-api-client`](/stacks/api/client) - API client library for all Stacks Blockchain API endpoints.
  </Accordion>
</Accordions>
