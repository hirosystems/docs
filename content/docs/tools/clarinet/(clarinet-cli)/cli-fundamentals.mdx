The Clarinet CLI provides a comprehensive suite of tools for Clarity smart contract development. From project initialization to deployment, Clarinet streamlines your entire development workflow.

## Core Commands

- Create a new project: [`clarinet new`](#clarinet-new)
- Generate a new smart contract: [`clarinet contracts new`](#clarinet-contracts-new)
- Validate contract syntax and types: [`clarinet check`](#clarinet-check)
- Interactive REPL for testing contracts: [`clarinet console`](#clarinet-console)
- Launch a local development network: [`clarinet devnet start`](#clarinet-devnet-start)
- Manage deployments: [`clarinet deployments`](#clarinet-deployments)

## Initialize a new project

Creates a new Clarinet project with all necessary configuration files and directory structure.

**Usage**

```terminal
$ clarinet new [OPTIONS] <NAME>
```

**Arguments**

- `<NAME>` Project's name

**Options**

| Option | Description |
|--------|-------------|
| `--disable-telemetry` | Do not provide developer usage telemetry for this project |

**Examples**

```terminal
$ clarinet new my-defi-protocol
[32mCreate directory[0m [1mmy-defi-protocol[0m
[32mCreate directory[0m [1mcontracts[0m
[32mCreate directory[0m [1mtests[0m
[32mCreate directory[0m [1msettings[0m
[32mCreate file[0m [1mClarinet.toml[0m
[32mCreate file[0m [1mpackage.json[0m
[32mCreate file[0m [1mvitest.config.js[0m
```

## Validate your contracts and explore with interactive REPL

Check contracts syntax and perform type checking.

**Usage**

```terminal
$ clarinet check [OPTIONS] [FILE]
```

**Arguments**

- `[FILE]` If specified, perform a simple syntax-check on just this one file

**Options**

| Option | Short | Description |
|--------|-------|-------------|
| `--manifest-path <path>` | `-m` | Path to Clarinet.toml |
| `--deployment-plan-path <path>` | `-p` | If specified, use this deployment file |
| `--use-on-disk-deployment-plan` | `-d` | Use on disk deployment plan (prevent updates computing) |
| `--use-computed-deployment-plan` | `-c` | Use computed deployment plan (will overwrite on disk version if any update) |
| `--enable-clarity-wasm` | | Allow the Clarity Wasm preview to run in parallel with the Clarity interpreter (beta) |

**Examples**

```terminal
$ clarinet check
[32m✔[0m [1m3 contracts checked[0m
```

Check a specific file:

```terminal
$ clarinet check contracts/token.clar
[32m✔[0m [1mcontracts/token.clar syntax checks passed[0m
```

With errors:

```terminal
$ clarinet check
[31m✗[0m [1m1 error in token.clar:[0m
  [33mLine 12:[0m Unknown function 'transfer-tokens'
  [90mDid you mean 'transfer'?[0m
```

---

Load contracts in a REPL for an interactive session.

**Usage**

```terminal
$ clarinet console [OPTIONS]
```

**Options**

| Option | Short | Description |
|--------|-------|-------------|
| `--manifest-path <path>` | `-m` | Path to Clarinet.toml |
| `--deployment-plan-path <path>` | `-p` | If specified, use this deployment file |
| `--use-on-disk-deployment-plan` | `-d` | Use on disk deployment plan (prevent updates computing) |
| `--use-computed-deployment-plan` | `-c` | Use computed deployment plan (will overwrite on disk version if any update) |
| `--enable-remote-data` | `-r` | Enable remote data fetching from mainnet or a testnet |
| `--remote-data-api-url <url>` | `-a` | Set a custom Stacks Blockchain API URL for remote data fetching |
| `--remote-data-initial-height <height>` | `-b` | Initial remote Stacks block height |
| `--enable-clarity-wasm` | | Allow the Clarity Wasm preview to run in parallel with the Clarity interpreter (beta) |

**Console Commands**

Once in the console, these special commands are available:

| Command | Description |
|---------|-------------|
| `::help` | Display all available console commands |
| `::list_contracts` | Show all deployed contracts |
| `::describe <contract>` | Display contract interface |
| `::get_assets_maps` | Show current STX balances |
| `::advance_chain_tip <number>` | Mine specified number of blocks |
| `::get_block_height` | Display current block height |
| `::set_epoch <version>` | Change active Stacks epoch |

**Examples**

```terminal
$ clarinet console
[32mclarity-repl v1.0.0[0m
Enter ".help" for usage hints.
Connected to a transient in-memory database.
>> (contract-call? .counter increment)
(ok true)
>> ::get_block_height
2
```

Using remote data from mainnet:

```terminal
$ clarinet console --enable-remote-data
[32mclarity-repl v1.0.0[0m
[33mRemote data enabled - mainnet[0m
Enter ".help" for usage hints.
>>
```

## Start a local development network

Subcommands for Devnet usage.

**Usage**

```terminal
$ clarinet devnet <COMMAND>
```

**Subcommands**

- `package` - Generate package of all required devnet artifacts
- `start` - Start a local Devnet network for interacting with your contracts from your browser

Start a local Devnet network for interacting with your contracts from your browser.

**Usage**

```terminal
$ clarinet devnet start [OPTIONS]
```

**Options**

| Option | Short | Description |
|--------|-------|-------------|
| `--manifest-path <path>` | `-m` | Path to Clarinet.toml |
| `--no-dashboard` | | Display streams of logs instead of terminal UI dashboard |
| `--deployment-plan-path <path>` | `-p` | If specified, use this deployment file |
| `--use-on-disk-deployment-plan` | `-d` | Use on disk deployment plan (prevent updates computing) |
| `--use-computed-deployment-plan` | `-c` | Use computed deployment plan (will overwrite on disk version if any update) |
| `--package <path>` | | Path to Package.json produced by 'clarinet devnet package' |

**Examples**

```terminal
$ clarinet devnet start
[32mStarting Devnet...[0m
[32m✔[0m Bitcoin node: http://localhost:18443
[32m✔[0m Stacks node: http://localhost:20443
[32m✔[0m Stacks API: http://localhost:3999
[32m✔[0m Explorer: http://localhost:8000
[33mOpening dashboard...[0m
```

Without dashboard UI:

```terminal
$ clarinet devnet start --no-dashboard
Starting Bitcoin node...
Starting Stacks node...
Devnet services running. Press Ctrl+C to stop.
```

## Manage your deployments

Manage contracts deployments on Simnet/Devnet/Testnet/Mainnet.

**Usage**

```terminal
$ clarinet deployments <COMMAND>
```

**Subcommands**

- `check` - Check deployments format
- `generate` - Generate new deployment
- `apply` - Apply deployment

---

Validate deployment files for syntax and configuration errors.

**Usage**

```terminal
$ clarinet deployments check [OPTIONS]
```

**Options**

| Option | Description |
|--------|-------------|
| `--manifest-path <path>` | Path to Clarinet.toml |

**Examples**

```terminal
$ clarinet deployments check
[32m✔[0m Deployment files are valid
```

---

Generate deployment plans for a specific network.

**Usage**

```terminal
$ clarinet deployments generate [OPTIONS]
```

**Options**

| Option | Description |
|--------|-------------|
| `--simnet` | Generate a deployment file for simnet environments (console, tests) |
| `--devnet` | Generate a deployment file for devnet, using settings/Devnet.toml |
| `--testnet` | Generate a deployment file for testnet, using settings/Testnet.toml |
| `--mainnet` | Generate a deployment file for mainnet, using settings/Mainnet.toml |
| `--manifest-path <path>` | Path to Clarinet.toml |
| `--no-batch` | Generate a deployment file without trying to batch transactions (simnet only) |
| `--low-cost` | Compute and set cost, using low priority (network connection required) |
| `--medium-cost` | Compute and set cost, using medium priority (network connection required) |
| `--high-cost` | Compute and set cost, using high priority (network connection required) |
| `--manual-cost` | Leave cost estimation manual |

**Examples**

```terminal
$ clarinet deployments generate --testnet
[32mGenerated deployment plan[0m
[32mCreated file[0m [1mdeployments/default.testnet-plan.yaml[0m
```

```terminal
$ clarinet deployments generate --mainnet --high-cost
[33mComputing deployment costs...[0m
[32mGenerated deployment plan with high priority fees[0m
[32mCreated file[0m [1mdeployments/default.mainnet-plan.yaml[0m
```

| `--deployment-plan-path <path>` | `-p` | Apply deployment plan specified |
| `--no-dashboard` | | Display streams of logs instead of terminal UI dashboard |
| `--use-on-disk-deployment-plan` | `-d` | Use on disk deployment plan (prevent updates computing) |
| `--use-computed-deployment-plan` | `-c` | Use computed deployment plan (will overwrite on disk version if any update) |

## Interact with Mainnet contracts

Interact with contracts deployed on Mainnet.

**Usage**

```terminal
$ clarinet requirements <COMMAND>
```

**Subcommands**

- `add` - Interact with contracts published on Mainnet

Add a mainnet contract as a dependency to your project.

**Usage**

```terminal
$ clarinet requirements add [OPTIONS] <CONTRACT_ID>
```

**Arguments**

- `<CONTRACT_ID>` - Contract id (ex. "SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.nft-trait")

**Options**

| Option | Description |
|--------|-------------|
| `--manifest-path <path>` | Path to Clarinet.toml |

**Examples**

```terminal
$ clarinet requirements add SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.nft-trait
[32mAdded requirement[0m SP2PABAF9FTAJYNFZH93XENAJ8FVY99RRM50D2JG9.nft-trait
[33mUpdated[0m [1mClarinet.toml[0m
```

#### Description

This command allows you to reference and use contracts that are already deployed on mainnet in your local development. This is useful for:
- Building on top of existing protocols
- Testing integrations with live contracts
- Importing standard traits and interfaces

## Editor Integration

### `clarinet lsp`

Get Clarity autocompletion and inline errors from your code editor (VSCode, vim, emacs, etc).

#### Usage

```terminal
$ clarinet lsp
```

#### Description

Starts the Language Server Protocol service for Clarity, enabling intelligent code completion, error highlighting, and other IDE features in supported editors.

### `clarinet dap`

Step by step debugging and breakpoints from your code editor (VSCode, vim, emacs, etc).

#### Usage

```terminal
$ clarinet dap
```

#### Description

Starts the Debug Adapter Protocol service, enabling debugging features like breakpoints, step-through execution, and variable inspection in supported editors.

## Utilities

### `clarinet completions`

Generate shell completion scripts for your shell.

#### Usage

```terminal
$ clarinet completions <SHELL>
```

#### Arguments

- `<SHELL>` - Specify which shell to generation completions script for

#### Supported Shells

- `bash`
- `zsh`
- `fish`
- `powershell`
- `elvish`

#### Examples

```terminal
$ clarinet completions zsh > ~/.zsh/completions/_clarinet
$ source ~/.zshrc
```

### `clarinet format`

Format Clarity code files according to standard conventions.

#### Usage

```terminal
$ clarinet format [OPTIONS]
```

#### Options

| Option | Short | Description |
|--------|-------|-------------|
| `--manifest-path <path>` | `-m` | Path to Clarinet.toml |
| `--file <file>` | `-f` | If specified, format only this file |
| `--max-line-length <length>` | `-l` | Maximum line length |
| `--indent <size>` | `-i` | Indentation size, e.g. 2 |
| `--tabs` | `-t` | Use tabs instead of spaces |
| `--dry-run` | | Only echo the result of formatting |
| `--in-place` | | Replace the contents of a file with the formatted code |

#### Examples

```terminal
$ clarinet format
[32m✔[0m Formatted 3 contracts
```

Format a specific file:

```terminal
$ clarinet format --file contracts/token.clar --in-place
[32m✔[0m Formatted contracts/token.clar
```

Preview formatting changes:

```terminal
$ clarinet format --file contracts/token.clar --dry-run
;; Formatted output:
(define-public (transfer (amount uint) (recipient principal))
  (ok true)
)
```

Custom formatting options:

```terminal
$ clarinet format --indent 4 --max-line-length 100
[32m✔[0m Formatted 3 contracts with custom settings
```

## Environment Variables

Clarinet supports environment variables for configuration. All environment variables are prefixed with `CLARINET_`:

```terminal
$ export CLARINET_MANIFEST_PATH=/path/to/project
```

<Callout type="tip" title="Pro Tip">
Use tab completion in your shell to discover commands and options. Install shell completions with `clarinet completions <shell>` where shell is bash, zsh, fish, powershell, or elvish.
</Callout>
