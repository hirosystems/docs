## Overview

The Clarity VS Code extension provides a rich development experience for writing Clarity smart contracts. It offers syntax highlighting, code completion, error checking, and more.

## Features

### Syntax Highlighting
- Full Clarity language support
- Semantic highlighting
- Bracket matching
- Code folding

### IntelliSense
- Auto-completion for:
  - Built-in functions
  - Contract functions
  - Variables and constants
  - Data types
- Parameter hints
- Function signatures

### Error Checking
- Real-time syntax validation
- Type checking
- Reference validation
- Warning annotations

### Code Navigation
- Go to definition
- Find all references
- Symbol search
- Outline view

### Debugging Support
- Breakpoint setting
- Step debugging
- Variable inspection
- Call stack view

## Installation

### From VS Code

1. Open VS Code
2. Press `Cmd+Shift+X` (macOS) or `Ctrl+Shift+X` (Windows/Linux)
3. Search for "Clarity"
4. Click Install on the official extension by Hiro

### From Command Line

```bash
code --install-extension hirosystems.clarity-lsp
```

## Configuration

### Extension Settings

Access settings through VS Code preferences:

```json
{
  "clarity.backend": "clarinet",
  "clarity.trace": "off",
  "clarity.maxNumberOfProblems": 100,
  "clarity.suggest.completeFunctionCalls": true
}
```

### Available Settings

| Setting | Description | Default |
|---------|-------------|---------|
| `clarity.backend` | LSP backend to use | `"clarinet"` |
| `clarity.trace` | LSP trace level | `"off"` |
| `clarity.maxNumberOfProblems` | Maximum diagnostics | `100` |
| `clarity.suggest.completeFunctionCalls` | Complete function calls | `true` |

## Usage

### Auto-completion

Start typing to see suggestions:

```clarity
(define-public (transfer)
  (ok (stx-tra  ;; Auto-completes to stx-transfer?
```

### Hover Information

Hover over functions for documentation:

```clarity
(stx-transfer? ;; Shows function signature and description
```

### Quick Fixes

Click the lightbulb icon for fixes:
- Add missing imports
- Fix typos in function names
- Correct type mismatches

### Code Formatting

Format your code with:
- Right-click → Format Document
- `Shift+Alt+F` (Windows/Linux)
- `Shift+Option+F` (macOS)

## Features in Detail

### Type Checking

The extension provides real-time type checking:

```clarity
;; Error: Type mismatch
(define-public (add (x int) (y uint))
  (ok (+ x y))) ;; Cannot add int and uint
```

### Contract References

Easily navigate between contracts:

```clarity
;; Ctrl+Click on contract name to jump to definition
(contract-call? .other-contract function-name)
```

### Snippets

Built-in code snippets for common patterns:

| Trigger | Expands to |
|---------|------------|
| `defpub` | Public function template |
| `defpriv` | Private function template |
| `defconst` | Constant definition |
| `defdata` | Data variable |
| `defmap` | Map definition |
| `deftrait` | Trait definition |

### Integrated Terminal

Run Clarinet commands without leaving VS Code:
- View → Terminal
- Execute `clarinet test`, `clarinet check`, etc.

## Troubleshooting

### Extension Not Working

1. Check Clarinet is installed:
   ```bash
   clarinet --version
   ```

2. Reload VS Code window:
   - `Cmd+Shift+P` → "Developer: Reload Window"

3. Check extension logs:
   - View → Output → Select "Clarity Language Server"

### Performance Issues

- Disable unused extensions
- Increase memory limit in settings
- Close unnecessary files

### Missing Features

Ensure you have:
- Latest VS Code version
- Latest extension version
- Clarinet updated

## Tips and Tricks

### Multi-Cursor Editing
- `Alt+Click` to add cursors
- Useful for bulk edits

### Code Folding
- Collapse functions for overview
- `Ctrl+Shift+[` to fold
- `Ctrl+Shift+]` to unfold

### Integrated Debugging
- Set breakpoints in test files
- Use debug console for inspection

### Custom Snippets
Create your own snippets:
1. File → Preferences → Configure User Snippets
2. Select "clarity"
3. Add custom templates

## Integration with Clarinet

The extension automatically detects Clarinet projects:
- Reads `Clarinet.toml`
- Provides project-wide IntelliSense
- Integrates with test runner
